# Library Policy

## Purpose
Provide a unified screen for generated dataset/model assets across projects.

## Access Policy
- Roles: `project_owner`, `data_developer`, `model_developer`, `admin`
- Read access for all listed roles

## UX Policy
- Default tab: `Dataset`
- Default sort: `Recently Added`
- View mode supports `list` and `grid`

## Data Policy
- Asset source values must match schema source options
- Empty state appears when no assets match current filters
